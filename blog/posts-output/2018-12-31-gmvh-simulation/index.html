A simulation of Fisherian evolution to investigate the Greater Male Variability Hypothesis (GMVH)

About

Hill and Tabachnikov (2018) had proposed an analytical model which predicts that high female choosiness favors males with high phenotypic variability. Here I present a computational model roughly based on the assumptions made in their paper and investigate 1) whether factors other than female choosiness can impact GMV fitness, and 2) whether and under which conditions there can exist Fisherian runaway selection on both female choosiness and the GMV genotype.

Requirements

Mac OS X requirements:

brew install numpy scipy
brew install gnuplot
brew install jq

I strongly recommend using a virtual environment for Python such as pyenv, since you will also need to install following packages from PyPi:

pip install matplotlib pandas tblib tqdm asteval pygam

Then,

git clone https://github.com/LocusOfCtrl/gmvh_sim
cd gmvh_sim

Running

Basics

First we need to know whether things work. We can find out by running a small simulation with default settings:

python simulation.py --generations 3 --seed 1234 | jq -c \
    '{"gen": .generation, "counts":.genotypes}'

where generations parameter is the number of generations to run and seed is random number generator seed for reproducibility (if not given a set value, the output will vary every time the program is run). The jq command simply filters out the output by .genotypes field. The output should be:

{"gen":0,"counts":{"1":181,"total":365,"2":184}}
{"gen":1,"counts":{"1":165,"total":381,"2":216}}
{"gen":2,"counts":{"1":149,"total":360,"2":211}}
{"gen":3,"counts":{"1":142,"total":359,"2":217}}
{"gen":"*","counts":{"1":[181,165,149,142],"total":[365,381,360,359],"2":[184,216,211,217]}}

The generation labeled as "0" represents the initial conditions. The last line (generation "*") is simply a summary of the simulation. Now let's see what happens when we let the simulation run for 100 generations:

 python simulation.py --generations 100 --seed 1234 | tail -n5 | head -n4 \
    | jq -c '.genotypes'

The output is:

{"1":252,"total":341,"2":89}
{"1":261,"total":358,"2":97}
{"1":271,"total":366,"2":95}
{"1":243,"total":340,"2":97}

The genotype "1" (associated with the less variable phenotype) appears to be winning with default settings. However, the trend is weak and could be due to genetic drift. To verify that the effect is not random, we would need to repeat the simulation a sufficient number of times.

Workflow 1 - time series plots and mating systems

Does the data fit the model?

Above, we ran a single instance of simulation.py. For anything interesting, we would need to run more than one instance. This can be sped up by using the xargs scheduler:

seq 10 | sed 's/\(.*\)/out\1.ndjson/g' | xargs -P 4 -n 1 \
    python simulation.py --generations 100 --num_pools 100 \
    --pool_size 32 --female_choosiness 0.6 --male_choosiness 0.2 \
    --output_dir out --output_file

This will create a directory called out, spawn 4 worker processes via xargs, and distribute tasks across those processes in the form of file names generated by the seq and sed commands. Because this workflow is very common, I have written a wrapper (conduct.py) which removes the need to use an external scheduler and which lets one specify groups of parameters in a config-like file. The following line is equivalent to the previous:

python conduct.py --experiment female.pref --output_dir out

where female.pref is defined as an entry in experiments.py that looks like this:

"female.pref": {
    # females are fairly choosy, males not so much
    "pool_size": 32,
    "num_pools": 100,
    "generations": 100,
    "qual_x_num_kids": 0.2,
    "female_choosiness": 0.5,
    "male_choosiness": 0.2,
    "seed": range(10)
},

Because seed argument above is an array of ten integers, ten simulations with identical initial conditions (except for the random number generator seed) will be repeated.

After the simulation is finished (using either method), we do some post-processing on the output:

jq -rc '[input_filename, .generation, .genotypes."2"/.genotypes."total"]
| @csv'  out/out*.ndjson* | python csvtool.py pivot --index 1 --columns 0 \
--values 2 --rename_columns > out/piv.csv

gnuplot -c plot/series.gnu out/piv.csv out/fig-female.pref.svg "generation"

There is a theoretical reason why this process follows a logistic curve. The slope of this curve is estimated from a stochastic model and stored in .delta.*.kl field of the last output object (which can be obtained using select(.generation=="*") filter). Let's see if we get a good fit:

gnuplot -c plot/series_fit.gnu out/piv.csv out/fig-female.pref-fit.svg \
    $( jq 'select(.generation=="*")|.delta."2".kl.est' out/out*.ndjson* | \
    awk '{a+=$0}END{print a/NR}' )

Evaluating fertility and mating systems

Different settings of female and male choosiness result in different types of mating systems. The following plots should help us determine which mating system we're dealing with:

jq -rc 'select(.generation=="*")' out/out*.ndjson* | python mating_stats.py \
    --outmales out/summary_males.csv --outfemales out/summary_females.csv

gnuplot -c plot/multi.gnu out/summary_males.csv out/fig-female.pref-males.svg \
    "partners per male"

The term "partners" plotted on the X axis refers to the number of partners with whom the male had attempted to produce offspring, so it's not directly translatable to "has had sex for leisure" (which may be one misinterpretation of it). The plot above looks like a type of facultative polygyny where a half of all males are never selected for mating, a fifth mate with one female, another fifth with two females, and large "harems" are quite rare. Note that at levels above one mating partner, male quality increases only slightly (with some regression toward the mean at the extreme end), while the males who have not been selected for mating have dramatically lower average quality compared even to the males who have only one partner. In popular culture, this is roughly known as the "incel" phenomenon.

If a similar situation holds in the real world, one could hypothesize here that male mating status should serve as a strong quality signal to other females (mated males are perceived as more desirable than non-mated ones).

gnuplot -c plot/multi.gnu out/summary_females.csv out/fig-female.pref-females.svg \
    "children per female"

Unlike with males, the simulation attempts to mate all females, though not all of them may be assigned any children (i.e. there may be a fair amount of childless females). The proportion of childless females can be adjusted with binom_failures parameter, which in human terms can be thought to model the social pressure to have children. The number of children assigned to a given female is weakly determined by female quality (the strength of the effect is controlled by qual_x_num_kids parameter).

Unsurprisingly, the left pane of the figure reproduces the negative binomial distribution, which is the distribution from which the number of children for each female is sampled in the simulation. This distribution gives sensible results under the conditions of high fertility (see this plot of 2013 Nigerian DHS data) as well as when fertility is at the replacement level.

What happens when nobody is choosy?

In this simulation, "choosiness" controls the hypergamous tendency to reject the best available partner. When choosiness is zero, mating is still assortative (within each pool) but with many more committed matches aka "marriages." We now model the situation where both sexes are not "choosy": 

python conduct.py --experiment model0 --output_dir out

jq -rc '[input_filename, .generation, .genotypes."2"/.genotypes."total"]
| @csv'  out/out*.ndjson* | python csvtool.py pivot --index 1 --columns 0 \
--values 2 --rename_columns > out/piv.csv

gnuplot -c plot/series_fit.gnu out/piv.csv out/fig-model0-fit.svg \
    $( jq 'select(.generation=="*")|.delta."2".kl.est' out/out*.ndjson* | \
    awk '{a+=$0}END{print a/NR}' )

By 100th generation, the GMV genotype is almost extinct.

jq -rc 'select(.generation=="*")' out/out*.ndjson* | python mating_stats.py \
    --outmales out/summary_males.csv --outfemales out/summary_females.csv

gnuplot -c plot/multi.gnu out/summary_males.csv out/fig-model0-males.svg \
    "partners per male"

The distribution of female frequencies and qualities vs number of children looks very similar to female.pref results described previously, so I do not show it. The left pane suggests that we have something similar to monogamy. By default, males are assigned somewhat longer reproductive lifespans compared to females (this is controlled by male_lifespan parameter). This means that a portion of males mate again in the subsequent generation, which explains the 20% or so of males with two partners. Note however that the average quality of "incel" males has not improved under monogamy—it even declined somewhat. This is because with more males being matched overall, this particular cohort of males is now sampled from a more extreme lower end of the quality distribution.

Workflow 2 - single effects

Effect of female choosiness

This is very similar to the above except we're profiling female choice instead of male choice (fixing the latter to 0.2):

mkdir out-female_choosiness
python conduct.py --experiment female_choosiness --male_choosiness 0.2 \
    --num_pools 100 | gzip > out-female_choosiness/out.ndjson.gz

gunzip -c out-female_choosiness/out.ndjson.gz | jq -rc \
    '[.args.seed, .args.female_choosiness, .delta."2".kl.est ] | @csv' \
    > out-female_choosiness/kl.csv

When female choosiness is high, the relative fitness of the variable genotype becomes vary variable. So ExpectileGAM is more appropriate here as it allows for non-constant variance:

python csvtool.py smooth --in out-female_choosiness/kl.csv \
    --columns 1 2 --no_out_header --model expectile --n_splines 12 \
    > out-female_choosiness/kl-smooth.csv

gnuplot -c plot/circles_gam.gnu out-female_choosiness/kl.csv \
    out-female_choosiness/kl-smooth.csv out-female_choosiness/fig-gam.svg \
    "female choosiness"

Effect of male choosiness

We will vary male choosiness while keeping female choosiness fixed at 0.4. There is already an experiment defined in experiments.py for this purpose:

mkdir out-male_choosiness
python conduct.py --experiment male_choosiness --female_choosiness 0.4 \
    --num_pools 100 | gzip > out-male_choosiness/out.ndjson.gz

gunzip -c out-male_choosiness/out.ndjson.gz | jq -rc \
    '[.args.seed, .args.male_choosiness, .delta."2".kl.est ] | @csv' \
    > out-male_choosiness/kl.csv

python csvtool.py smooth --in out-male_choosiness/kl.csv \
    --columns 1 2 --no_out_header --model linear --n_splines 10 \
    > out-male_choosiness/kl-smooth.csv

gnuplot -c plot/circles_gam.gnu out-male_choosiness/kl.csv \
    out-male_choosiness/kl-smooth.csv out-male_choosiness/fig-gam.svg \
    "male choosiness"

Above I used LinearGAM because the variance seems roughly constant across different effect sizes, something that is not the case with female choosiness.

The plot shows an interesting effect: there is a peak in relative fitness favoring the GMV genotype when male choosiness is close to 0.8. In this section I look into why this is the case.

Other effects

To generate plots of all single effects supported by the simulation:

mkdir out-single

python conduct.py --num_pools 100 --experiment . | gzip \
    > out-single/out.ndjson.gz

python make_plots.py out-single

I do not show the figures on this page as there are too many, but you can visit the index page that lists them.

Workflow 3 - analysis of interactions

Change in prevalence

The mating process in each pool is competitive, i.e. males of one genotype get to compete with males of another genotype, as well as with each other. The choosiness of a given individual is weakly affected by the quality of its phenotype (the extent of this effect can be controlled with qual_x_choosiness parameter). It may be interesting to see what the GMV fitness looks like given varying levels of female and male choosiness. In other words, we would like to model the interaction between male and female choosiness. To do this, we configure an experiment labeled fcmc like so:

"fcmc": {
    "inheritance_pattern": "male",
    "g2_male_variability": 1.5,
    "g2_female_variability": 1.0,
    "female_choosiness": np.linspace(0.0, 1.0, 21),
    "male_choosiness": np.linspace(0.0, 1.0, 21),
    "seed": range(30),
    "output_level": 0
},

There are three fields that have 1D arrays assigned to them: female_choosiness, male_choosiness, and seed. The first two are the parameters of interest, while the third one is essentially the number of samples. This configuration produces a 3D parameter grid with 13,230 entries (21 * 21 * 30 = 13,230). The larger the grid, the longer it will take to run the entire setup. We run the simulation like so:

mkdir out-fcmc
python conduct.py --experiment fcmc --qual_x_num_kids 0.2 --num_pools 100 \
    | gzip > out-fcmc/out.ndjson.gz

The qual_x_num_kids and num_pools command-line argument overrides the default settings of 0.1 and 5, respectively. Lower number of pools results in faster run time at the expense of lower accuracy. Note that there is an essential difference between two pool parameters: num_pools has almost no effect on the simulation, while large pool_size produces very efficient matchings between individuals of opposite sex (aka "large city + Tinder" situation). When pool_size is small, the simulation models the conditions of small villages/groups, which were probably found throughout most of human history.

Let's see how the GMV genotype (labeled by the simulation as genotype "2") does over the course of the simulation. To do this, we take its last observed frequency, subtract from it the initial frequency of 0.5, and scale the result such that +100 represents 100% increase in frequency (i.e. complete takeover) and -100 represents 100% decline (i.e. extinction):

gunzip -c  out-fcmc/out.ndjson.gz | jq -rc 'select(.generation=="*") |[.args.male_choosiness,
.args.female_choosiness, .genotypes."2"[-1], .genotypes."total"[-1]] | @csv' \
| python csvtool.py aggregate --no_out_header --keys 0 1 --function sum \
    > out-fcmc/out-p2.csv

bash plot/3d.sh out-fcmc/out-p2.csv '200*($3/$4-.5)' 0.06 "male choosiness" \
"female choosiness" "% change in prevalence" out-fcmc/fig-fcmc-p2-3d.svg

There is a nearly complete takeover by the GMV genotype as female choosiness approaches 1.0. The populations inside inside the bottom-left contour experienced slight negative selection against the variable male genotype (its prevalence decreased by approximately 20%), while populations almost everywhere outside of that zone experienced positive selection.

The fitness landscape of the variable male genotype

The change in prevalence may be easy to interpret, but it does not measure evolutionary pressure accurately. To estimate the latter, it helps to think in terms of growth rates. Suppose we have two subpopulations 1 and 2 growing exponentially at rates k1 and k2. Since the simulation ensures that the overall number individuals remains stable over time (replacement-level reproduction), we can only observe the difference in growth rates d = k2 - k1, which is also the slope of the logistic curve shown previously. This difference tells us how much better the GMV genotype ("2") is doing with respect to baseline genotype ("1") under competitive mating.

The difference in growth rates d represents a measure of evolutionary pressure. If we take the exponent of d, we get relative fitness of genotype 2 with respect to genotype 1. Thus, when d is zero, the relative fitness of g2 with respect to g1 is equal to 1.0 (meaning the fitness levels of g2 and g1 are the same). I will refer to d as log relative fitness.

Once we have a single number that represents relative fitness, we can make more complicated analyses such as the one discussed here. The estimated value of d is stored inside .delta field in the last JSON object in the output. Let's plot how the values of d for the variable male genotype vary with female and male choosiness:

gunzip -c out-fcmc/out.ndjson.gz | jq -rc \
'[.args.male_choosiness, .args.female_choosiness, .delta."2".kl.est] | @csv' \
| python csvtool.py smooth3d --n_splines 10 10 12 12 --no_out_header \
    > out-fcmc/fcmc-kl-smooth.csv

The smoothing step fits an expectile GAM to the collected d estimates. Further smoothing is applied during plotting through simple gaussian blur of radius 0.02 (the size of the radius is best determined by eyeballing the contours):

bash plot/3d.sh out-fcmc/fcmc-kl-smooth.csv '$3' 0.02 "male choosiness" \
"female choosiness" "log relative fitness" out-fcmc/fig-fcmc-kl-3d.svg

This looks much flatter than the previous plot. The ridge when male choosiness is close to 0.8 has almost disappeared. The reason for this should become obvious after considering the slope of the logistic function which models evolution rate in our case: the slope is highest when y ~ 0.5. This means that when discrepancy in fitness levels is large and population is split equally between two competing genotypes, the population approaches the condition of genetic homogeneity very quickly, after which the absolute rate of selection slows down significantly. Notably, this is precisely Fisher's fundamental theorem of natural selection: the rate of increase in fitness of the population (roughly: the rate of evolution) is equal to the genetic variance of the population (which, in the simple binomial case, is expressed as pq and reaches a maximum when p = q = 0.5).

In other words, depending on the initial conditions and the length of the time period, even small discrepancies in fitness may have significant impact. To better visualize how d varies depending on the two traits of interest, we construct a fitness landscape, which is essentially a 2D contour plot of the 3D figure above:

gnuplot -c plot/contours.gnu out-fcmc/fcmc-kl-smooth.csv \
out-fcmc/fig-fcmc-kl-contours.svg 0.02 "male choosiness" "female choosiness"

It may be interesting to gauge the inequality of sexual outcomes among males (as measured by the Gini coefficient on the number of partners):

gunzip -c out-fcmc/out.ndjson.gz | python annotate.py --gini | jq -rc \
'[.args.male_choosiness, .args.female_choosiness, .malequal_x_matches.gini] | @csv' \
| python csvtool.py smooth3d --n_splines 12 12 12 12 --no_out_header \
> out-fcmc/fcmc-gini-smooth.csv

gnuplot -c plot/contours.gnu out-fcmc/fcmc-gini-smooth.csv \
out-fcmc/fig-fcmc-gini-contours.svg 0.02 "male choosiness" "female choosiness" 0 1

Why does male choosiness matter?

In most species where sexual selection is present, it is driven by female choice as opposed to male choice. Nevertheless, it is interesting to consider the case of high male choosiness, if only to understand why it affects the fitness of GMV males at all. A first guess might be that high male choosiness leads to more polygynous societies since it will increase the number of reluctant uncommitted males, and the females that would otherwise be paired to those males end up becoming single mothers while mating with higher quality males from the upper percentile(s) instead. This is not exactly wrong, but it doesn't explain why selection for the GMV genotype then declines as male choosiness reaches its maximum.

It turns out that male choosiness affects sorting of males by quality, and the shape of the resulting distribution affects selection for the variable male genotype. For example, consider female.pref (shown previously) and male.pref models. In both, the variable male genotype has similar relative fitness. Assuming the data for these experiments is in out-female.pref and out-male.pref, we can compare the mating systems with respect to males:

gnuplot -c plot/compare_mating.gnu out-female.pref/summary_males.csv \
out-male.pref/summary_males.csv fig-compare_prefs.svg "partners per male" \
"female.pref" "male.pref"

The mating system under stronger female preference is actually more polygynous (left pane). What matters for the GMV fitness, however, is whether high-quality males are mated more frequently, as well as whether low-quality males are consistently rejected (right pane). Raising male choosiness further to its maximum value of 1.0 results in even more benefit to high-quality males, however the slope of their quality distribution becomes more linear (not shown), that is low-quality males are no longer doing as badly (relative to medium-quality ones).

Why is this interesting? Because it's conceivable, in my opinion, that the mating/sorting system we are seeing when male choosiness exceeds female could be established through mechanisms other than through male choice – for example, through arranged marriage. To summarize, when choosiness by either sex is high, there is a variety of polygynous and semi-polygynous mating systems that lead to greater relative GMV fitness (with female choice having by far the greatest effect), but only near-monogamy has a stabilizing effect on males.

Fisherian runaway selection towards polygyny and male variability

It is interesting to try to determine under what conditions the two main traits analyzed above (female choosiness and male variability) can co-evolve. Assuming that both are quantitative traits, we investigate possible covariance between the two:

mkdir out-mvar
python conduct.py --experiment mvar --num_pools 100 | gzip > out-mvar/out.ndjson.gz

This is very similar to fcmc except instead of varying male choosiness, we vary the variability effect exerted by the GMV genotype by setting "g2_male_variability": np.linspace(1.0, 2.0, 21) while keeping male choosiness fixed at 0.2.

gunzip -c out.ndjson.gz | jq -rc \
'[.args.g2_male_variability, .args.female_choosiness, .delta."2".kl.est] | @csv' \
| python csvtool.py smooth3d --n_splines 12 12 12 12 --no_out_header \
    > out-mvar/mvar-kl-smooth.csv

bash plot/3d.sh out-mvar/mvar-kl-smooth.csv '$3' 0.02 "g2 male variability" \
"female choosiness" "log relative fitness" fig-mvar-kl.svg

Here is a contour plot of the above (the Z-axis is clipped to 0.8):

gnuplot -c plot/contours.gnu out-mvar/mvar-kl-smooth.csv \
out-fcmc/fig-mvar-contours.svg 0.06 "g2 male variability" "female choosiness"

That the two traits (female choosiness and male variability) are covariant is obvious from the convex contour of the fitness landscape above y ~ 0.45. Below that threshold, the convexity disappears, meaning that the two traits become independent. Above the zero contour, however, conditions are ripe for Fisherian runaway selection towards the upper-right corner. The selection gradient over the two traits can be visualized as a vector field:

gnuplot -c plot/vectors.gnu out-mvar/mvar-kl-smooth.csv \
out-fcmc/fig-mvar-vectors.svg 0.06 "g2 male variability" "female choosiness"

The zero contour above demarcates a boundary between runaway selection on both traits in the upper-right direction and stabilizing selection on female choosiness (notice small arrows pointing downward at around y ~ 0.45). Because in our model choosiness represents willingness to reject the best available (unpaired) mate, the contour at y ~ 0.45 can be thought of as polygyny threshold.

Effect of parental investment

By "parental investment" here I refer how much parental phenotype quality (in case of "single mothers" this turns into maternal phenotype quality) affects offspring phenotype quality. This effect ranges from 0.0 (no connection) to 1.0 (offspring phenotype quality is fully determined by that of the parents). In case of marriages, the parents contribute equally towards the quality of their children, and their resources are "split" between offspring s.t. children in large families are penalized. In case of "single mothers," the contribution is only from the mother.

mkdir out-fcpi
python conduct.py --experiment fcpi --num_pools 100 | gzip > out-fcpi/out.ndjson.gz

python make_plots.py --experiment fcpi out-fcpi

This shows there exist a threshold near midpoint of the X axis s.t. at low values of parental investment and high female choosiness, the variable male genotype is more strongly selected for than at high values of parental investment and high female choosiness. When female choosiness is low, interestingly, the effect is inverted, albeit with much weaker selection in either direction.
